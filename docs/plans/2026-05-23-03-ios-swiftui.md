# iOS (SwiftUI, v2) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task.
> **Status: RE-GRAINED FOR EXECUTION.** Outline expanded into TDD-friendly tasks (~30–90 min each). User has explicitly overridden the "wait 2–4 weeks after web v1" recommendation; native development starts immediately in parallel with web v1 settling. Plan 04 (Android) is dispatched to a sibling agent — this plan owns only `ios/`.

**Goal:** Native iOS app that mirrors the seven Cardstock-designed screens shipped on web, consuming the Supabase + edge-function surface defined in plan 01. Native iOS chrome (tab bar, sheets, swipe gestures, haptics) layered over Cardstock visual identity.

**Architecture:** SwiftUI + vanilla MVVM (`@Observable` view models, no TCA for v2 to minimize learning curve). `supabase-swift` SDK for auth + table queries + RPCs (`similar_games`, `profile_candidates`). `swift-openapi-generator` for typed edge-function clients (`bgg-sync`, `resize-cover`). Cardstock theme tokens delivered via an `EnvironmentValue`. The 12-axis scoring engine is ported to Swift and runs client-side for lens re-ranking (zero roundtrip per lens change).

**Tech Stack:** Swift 6, SwiftUI, iOS 17+, Xcode 16+, `supabase-swift` 2.x, `swift-openapi-generator` build plugin, `Nuke` for image caching, `sentry-cocoa`, `posthog-ios`, Sign in with Apple, fastlane (Match for code signing).

---

## Scope cuts (locked)

1. **iPhone only for v2.** iPad gets a scaled iPhone layout (no iPad-specific sidebar/multi-column). Re-evaluate post-launch.
2. **No widgets, Live Activities, watchOS, or push notifications in v2.**
3. **No offline mode.** Network required; we show a Cardstock-styled "no connection" state.
4. **Sign in with Apple is the primary auth.** Google OAuth + magic link work via `ASWebAuthenticationSession`. Email/password not exposed in v2 mobile UI (parity gap with web is acceptable; magic-link covers it).
5. **No password reset surface.** Same rationale as web.
6. **Universal links for magic-link callback only.** No deep links into specific games in v2.

If you disagree with any of these, raise before Task 1 begins.

**Files:**

- Modify: `ios/yournextbg/Info.plist` (read `SUPABASE_URL` + `SUPABASE_ANON_KEY` from xcconfig)
- Modify: `ios/yournextbg.xcconfig`

```swift
import Supabase

enum SupabaseService {
    static let client: SupabaseClient = {
        guard let url = Bundle.main.object(forInfoDictionaryKey: "SUPABASE_URL") as? String,
              let key = Bundle.main.object(forInfoDictionaryKey: "SUPABASE_ANON_KEY") as? String,
              let supabaseURL = URL(string: url) else {
            fatalError("Supabase config missing from Info.plist")
        }
        return SupabaseClient(supabaseURL: supabaseURL, supabaseKey: key)
    }()
}
```

Service-role key is **never** in the iOS bundle. xcconfig committed without secrets; CI injects them.

**Test:** unit test that asserts the singleton initializes (mock `Info.plist` via a test bundle).

**Acceptance:** Build + tests pass.

| # | Risk | Mitigation |
|---|---|---|
| 1 | SIWA Service ID + Supabase OAuth provider not pre-configured | Phase 0 stops on this. Don't build the auth UI until SIWA exchanges to a real Supabase session manually. |

| 3 | Scoring engine drift between TS + Swift | Parity fixture (Task 3.2) is the source of truth. CI runs parity tests every build. |
| 4 | fastlane Match repo not yet provisioned | Phase 8 stops on this. Until then, builds are local-only with manual signing. |
| 5 | TestFlight review for first build is slow (24-48h) | Submit a stub build during Phase 0 to warm the review pipeline. (Decision deferred to user.) |
