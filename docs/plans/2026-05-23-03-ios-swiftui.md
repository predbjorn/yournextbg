# iOS (SwiftUI, v2) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task.
> **Status: OUTLINE.** This plan is intentionally less granular than plans 01 + 02 because iOS is v2 — to be started after web v1 has been live for 2–4 weeks and we have usage data + OpenAPI spec stabilized. Re-grain into bite-sized TDD tasks at that point.

**Goal:** Native iOS app that mirrors the seven web screens, consuming the Supabase + edge-function surface defined in plan 01. Cardstock design preserved with iOS-native chrome (tab bar, sheets, swipe gestures, haptics).

**Architecture:** SwiftUI + The Composable Architecture pattern (or vanilla MVVM if team is small). `supabase-swift` SDK for auth + tables + RPCs. `swift-openapi-generator` produces Swift clients from `contract/openapi.yaml` for edge-function calls. Theme tokens delivered via an `Environment` value.

**Tech Stack:** Swift 6, SwiftUI, iOS 17+, `supabase-swift`, `swift-openapi-generator`, `Nuke` for image caching, `Sentry-iOS`, `PostHogIOS`, `Sign in with Apple`, fastlane for App Store delivery.

## Phase 0 — Project setup

### Task 1: Xcode project + monorepo layout

- Create `ios/` directory at the monorepo root.
- `cd ios && xcrun ... Project new` (or generate from a template). Bundle id `com.yournextbg.app`. iOS 17+ minimum.
- Add SPM dependencies:
  - `https://github.com/supabase/supabase-swift` (2.x)
  - `https://github.com/apple/swift-openapi-generator` (build-time plugin)
  - `https://github.com/apple/swift-openapi-runtime`
  - `https://github.com/kean/Nuke` (image cache)
  - `https://github.com/getsentry/sentry-cocoa`
  - `https://github.com/PostHog/posthog-ios`
- Verify build: `xcodebuild -scheme yournextbg build`

### Task 2: OpenAPI client generation

- Add `swift-openapi-generator` build plugin pointing at `../contract/openapi.yaml`.
- Run a build; verify generated types appear under `.build/.../GeneratedSources/`.
- Hand-port table types (`Game`, `UserGame`, `UserPrefs`) into a Swift module `Sources/Models/` since Supabase table reads aren't auto-typed in Swift.

### Task 3: Theme environment

- Create `Sources/Theme/Theme.swift` mirroring the web's CSS-var theme tokens.

```swift
struct Theme {
    let paper: Color
    let paperWarm: Color
    let paperDeep: Color
    let ink: Color
    let inkSoft: Color
    let muted: Color
    let branchThinking: Color
    let branchInteraction: Color
    let branchLuck: Color
    let branchExperience: Color

    static let light = Theme(
        paper: Color(hex: 0xEFE6D0),
        // ... mirror prod-system.jsx LIGHT_TOKENS
    )
    static let dark = Theme(/* mirror DARK_TOKENS */)
}

private struct ThemeKey: EnvironmentKey {
    static let defaultValue = Theme.light
}

extension EnvironmentValues {
    var theme: Theme {
        get { self[ThemeKey.self] }
        set { self[ThemeKey.self] = newValue }
    }
}
```

- Wire to system `@Environment(\.colorScheme)` so light/dark follows the OS unless user_prefs says otherwise.

### Task 4: Supabase client singleton

- `Sources/Services/Supabase.swift`:

```swift
import Supabase

enum SupabaseService {
    static let client = SupabaseClient(
        supabaseURL: URL(string: "https://gkickjaihbgapowsqwhx.supabase.co")!,
        supabaseKey: Bundle.main.infoDictionary?["SUPABASE_ANON_KEY"] as! String,
    )
}
```

- Store anon key in `Info.plist` via `xcconfig` (not committed). Service-role key is **never** in the iOS bundle.

## Phase 1 — Auth

### Task 5: Sign in with Apple

- `SignInWithAppleButton(.signIn)` → on completion, exchange the Apple identity token with Supabase:

```swift
let token = credential.identityToken!.string
try await SupabaseService.client.auth.signInWithIdToken(
    credentials: .init(provider: .apple, idToken: token)
)
```

- Persist session via `supabase-swift`'s default `UserDefaults` storage (verify Keychain-equivalent is used in current SDK; if not, swap).

## Phase 3 — Screens

### Task 9: Shelf

- `LazyVGrid(columns: [.adaptive(minimum: 140)])` of `ShelfCard`.
- Pull-to-refresh re-fetches `collection_items`.
- `+ Add` toolbar button → modal sheet with typeahead search hitting `games` table.

### Task 10: Rate flow

- Card stack via `ZStack` with offsets/rotations matching `prod-mobile.jsx` `MobileRate`.
- Gesture: `DragGesture` → swipe-left = skip, swipe-up = detail, swipe-down = dismiss session.
- Tap a rating stamp button or use system haptics on submit (`UIImpactFeedbackGenerator(style: .medium)`).
- Optimistic update: pop the top card immediately, await the network call in background.

### Task 11: Recommendations

- `List` of `ProfileRecCard` rows.
- "Compare in lens" action navigates to `LensScreen(initialAnchor: .you, initialB: rec.id)`.
- "Dismiss" swipe action: `swipeActions(edge: .trailing)`.

### Task 12: Lens (the showpiece)

- Vertical stack per design's mobile lens: header → lens chips (horizontal `ScrollView`) → Game A picker → swap button → Game B picker → radar → similar list.
- Radar = custom SwiftUI `Canvas` view drawing 12 spokes and two polygons. Reuse the math (port from `src/lib/scoring/`).
- Lens chip change = recompute ranking locally (no roundtrip). Verify <16ms on midrange devices.

### Task 13: Game detail

- Hero image via `LazyImage` (Nuke).
- 4 `BranchBlock` views below.
- "If you like Brass…" 5-row horizontal `ScrollView`.

### Task 14: Profile / Settings

- `Form` with sections: Recommendation engine, Account.
- Dark mode toggle writes to `user_prefs.theme`.
- Sign out button.
- "Delete account" with confirmation alert.

## Open decisions to revisit at v2 kickoff

1. **TCA or vanilla MVVM?** TCA has steeper learning curve but better testability for the rate flow's state machine. Vanilla is faster to ship if you're solo on iOS.
2. **Universal app or iPad-specific layouts?** v2 default is iPhone-only; iPad gets a "scaled" experience. Re-evaluate based on user requests.
3. **Widgets, Live Activities, watchOS?** All out of scope for v2 initial release.
4. **App Store screenshots + marketing copy** — need design work; cardstock visual identity travels well to the screenshot frames.
